# Implemented User Flows

## OPD Flow

1. Reception registers or searches a patient in `/reception`.
2. Reception checks the patient in through `POST /visits`, creating a visit number and queue number.
3. Reception generates an OPD invoice and collects payment.
4. Nurse opens `/nursing`, selects the queued visit, records triage and vitals.
5. Emergency or critical vitals create doctor notifications automatically.
6. Doctor opens `/doctor`, writes SOAP notes, diagnosis, prescription, lab orders, or imaging orders.
7. Pharmacy, lab, and radiology receive their respective queue records.
8. Doctor can admit or discharge using clinical endpoints.

## Emergency Flow

1. Nurse creates or receives an emergency visit.
2. Triage category `resuscitation` or `emergent` escalates a critical notification to doctors.
3. Emergency visits can use deferred payment status.
4. Doctor can create STAT lab/imaging orders and prescriptions in the same consultation screen.

## Pharmacy Flow

1. Doctor submits eRx through `/clinical/visits/:visitId/prescriptions`.
2. Pharmacist sees the pending prescription in `/pharmacy`.
3. Pharmacist reviews allergy/interaction flags and dispenses.
4. Stock quantity is decremented when a batch is selected.
5. Low-stock notifications are generated for manager follow-up.

## Lab Flow

1. Doctor orders lab tests.
2. Lab user sees requests in `/lab`.
3. Lab collects sample, generating `LAB-...` sample code.
4. Lab enters structured analyte results and flags high/low/critical values.
5. Critical values notify the ordering doctor.
6. Lab validates and authorizes results.

## Radiology Flow

1. Doctor orders imaging.
2. Radiology user sees requests in `/radiology`.
3. Radiology captures procedure details and releases report/image URLs.
4. Urgent findings notify the ordering doctor.

## Director Flow

1. Director opens `/director`.
2. Dashboard shows daily patients, admissions, revenue, pending bills, bed occupancy, pending lab requests, department volume, top diagnoses, and stock alerts.
3. Director creates staff user accounts for reception, nurse, doctor, pharmacy, laboratory, radiology, manager, and director roles.
4. Director can add the onboarded user to the staff registry with employment details.
5. Director can deactivate staff who no longer work with the hospital and disable their linked platform login.
6. Director can activate, suspend, unlock, or reset staff user passwords.
7. Director can review audit logs generated by protected API activity.

## Manager Flow

1. Manager opens `/management`.
2. Finance shows monthly revenue, expenses, net position, daily billing summary, pending invoices, and recent expenses.
3. Manager records expenses for salaries, drugs and consumables, utilities, maintenance, vendor payments, and miscellaneous spend.
4. Manager maintains the staff registry for employment records, contact details, professional registration details, and optional platform access.
5. Manager filters and pages through large staff, invoice, expense, bed, and asset records.
6. Manager maintains the bed board and asset register for operational readiness.

## Security Flow

1. User signs in with email/password.
2. API enforces bcrypt password verification, failed-login lockout, JWT access token, refresh token rotation, and API-level role checks.
3. Protected mutations are logged to `AuditLog`.
4. Sensitive clinical assessment text is encrypted at field level before storage.

## Current Phase Notes

This implementation establishes the foundation and the core cross-department OPD workflow. The schema already includes expansion points for maternity, paediatrics, procurement, controlled substances, backups, integration settings, and external payment/SMS/NHIA providers.
